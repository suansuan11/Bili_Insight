# Bili-Insight 项目技术文档

## 1. 文档目的

本文档用于系统化总结 Bili-Insight 智能舆情分析平台的项目背景、功能模块、系统架构、核心业务流程、数据库设计、接口实现、算法方案、部署运行方式及当前实现状态。该文档可直接作为毕业论文初稿撰写的事实依据，也可作为项目答辩和系统说明材料。

## 2. 项目简介

### 2.1 项目名称

Bili-Insight 智能舆情分析平台

### 2.2 项目背景

随着 Bilibili 平台视频内容生态快速发展，视频评论和弹幕已经成为用户反馈的重要载体。对 UP 主而言，评论和弹幕中蕴含着视频节奏、内容质量、剪辑效果和观众接受度等关键信息；对品牌方而言，评论区与弹幕区则反映了新品口碑、投放效果和潜在负面舆情风险。因此，构建一个能够自动完成数据采集、情感分析、切面识别和可视化展示的 B 站舆情分析系统，具备较强的实际应用价值和工程实现价值。

### 2.3 项目目标

本项目旨在实现一个面向 B 站视频场景的舆情分析平台，完成以下目标：

1. 支持用户提交 BVID 进行视频级舆情分析。
2. 自动抓取视频评论、弹幕和视频基础信息。
3. 对评论与弹幕执行中文情感分析，并输出可解释的情感标签与分数。
4. 对评论执行切面级情感分析，识别用户讨论焦点。
5. 生成情绪时间轴，辅助定位视频高光与争议片段。
6. 提供热门视频榜单，为内容发现和分析入口服务。
7. 提供 B 站扫码绑定能力，提高部分分析接口稳定性。
8. 提供项目化监测、仪表盘和导出能力，形成完整业务闭环。

## 3. 系统整体架构

### 3.1 架构模式

系统采用“三层解耦”架构：

1. 前端展示层：Vue 3 单页应用。
2. Java 业务后端：Spring Boot。
3. Python 分析微服务：FastAPI。

这种架构将用户管理、任务编排、业务接口和文本分析能力进行解耦，既减少了单体系统复杂度，也便于后续按模块扩展。

### 3.2 分层职责

#### 3.2.1 前端

前端负责：

1. 用户登录注册界面。
2. 仪表盘展示。
3. 热门视频榜单展示。
4. 分析任务提交与轮询。
5. 分析详情页的评论、弹幕、时间轴和切面展示。
6. 项目管理页面。
7. 设置页与 B 站扫码绑定交互。

#### 3.2.2 Java 后端

Java 后端负责：

1. 用户认证与 JWT 鉴权。
2. 请求转发与接口聚合。
3. 分析任务记录与状态查询。
4. 项目管理、仪表盘统计、评论导出。
5. 调用 Python 分析服务。
6. 在系统启动时尝试拉起 Python 服务并触发热门视频初始化。

#### 3.2.3 Python 微服务

Python 服务负责：

1. 调用 Bilibili API 抓取视频信息、评论、弹幕和热门视频。
2. 执行中文情感分析和切面情感分析。
3. 生成情绪时间轴。
4. 将评论、弹幕、时间轴和热门视频等结果直接写入 MySQL。
5. 提供扫码登录二维码与轮询接口。

## 5. 系统功能模块

### 5.1 用户与认证模块

对应接口：

1. `/insight/auth/register`
2. `/insight/auth/login`
3. `/insight/auth/me`

功能说明：

1. 用户可以注册和登录平台账号。
2. 登录成功后后端签发 JWT。
3. 前端将 JWT 保存到本地，并通过路由守卫拦截未登录访问。

### 5.2 B 站扫码绑定模块

Java 侧接口：

1. `/insight/login/qrcode/generate`
2. `/insight/login/qrcode/poll`
3. `/insight/login/status/check`
4. `/insight/login/current_user`
5. `/insight/login/logout`

Python 侧接口：

1. `/api/login/qrcode`
2. `/api/login/status/{qrcode_key}`

业务逻辑：

1. 前端请求二维码。
2. Java 后端调用 Python 服务生成二维码数据。
3. 前端轮询扫码状态。
4. 确认登录后，Java 后端将 `SESSDATA`、`bili_jct`、`buvid3` 保存到当前用户记录。

### 5.3 分析任务模块

Java 侧分析接口：

1. `/insight/analysis/submit`
2. `/insight/analysis/status/{taskId}`
3. `/insight/analysis/result/{taskId}`
4. `/insight/analysis/comments/{taskId}`
5. `/insight/analysis/danmakus/{taskId}`
6. `/insight/analysis/timeline/{taskId}`
7. `/insight/analysis/recent`

Python 侧分析接口：

1. `/api/analysis/video`
2. `/api/analysis/status/{task_id}`
3. `/api/analysis/result/{task_id}`
4. `/api/analysis/timeline/{task_id}`
5. `/api/analysis/aspects/{task_id}`

业务逻辑：

1. 用户在前端提交 BVID。
2. Java 后端生成分析任务并写入 `analysis_task`。
3. Java 调用 Python 接口提交任务。
4. Python 后台异步执行抓取、分析和写库。
5. 前端通过状态接口轮询进度并展示结果。

### 5.4 热门视频模块

Java 侧接口：

1. `/insight/popular-videos`
2. `/insight/popular-videos/fetch`
3. `/insight/popular-videos/fetch/status`

Python 侧接口：

1. `/api/popular/fetch`
2. `/api/popular/fetch/status`
3. `/api/popular/list`

当前实现特点：

1. 热门视频抓取任务以后台任务形式执行。
2. 抓取逻辑会尽量收集至少 100 个唯一 `bvid`。
3. 只有在唯一热门视频数达到 100 时，才会清空旧表并重建新表。
4. 成功执行后，`popular_videos` 表中应保留 100 条热门视频记录。

### 5.5 项目监测模块

## 7. 核心业务流程

### 7.1 视频分析主流程

1. 用户在前端输入 BVID，提交分析任务。
2. Java 后端校验用户身份，并创建任务记录。
3. Java 调用 Python `/api/analysis/video` 接口。
4. Python 后台依次执行：
   - 获取视频信息
   - 获取评论
   - 获取弹幕
   - 评论情感分析与切面分析
   - 弹幕情感分析
   - 时间轴生成
   - 结果写库
5. Java 和前端通过任务状态接口获取进度。
6. 分析完成后前端展示完整结果。

### 7.2 B 站扫码绑定流程

1. 前端请求二维码。
2. Java 转发到 Python 登录接口。
3. Python 返回二维码链接与二维码 key。
4. 前端展示二维码并轮询状态。
5. 用户在 B 站客户端确认登录。
6. Java 在轮询确认成功后保存 B 站凭证到当前平台用户记录。

### 7.3 热门视频刷新流程

1. 前端或系统启动时触发热门视频刷新。
2. Python 后台抓取多页热门视频。
3. 根据 `bvid` 去重，直到凑够至少 100 个唯一视频。
4. 若不足 100，则任务失败并保留旧数据。
5. 若达到 100，则清空 `popular_videos` 表并插入 100 条新数据。

## 9. 情感分析与切面分析实现

### 9.1 初始方案问题

项目初始阶段使用 `SnowNLP + 阈值 + 单关键词切面` 的方案，存在以下问题：

1. 对 B 站口语化短文本适应性差。
2. 对反讽、网络语、缩写识别能力弱。
3. 只能提供粗粒度三分类结果。
4. 切面分析接近“关键词命中”，无法表达多切面混合情绪。

### 9.2 当前升级方案

当前 Python 分析链路已升级为：

1. 文本标准化。
2. Transformer 情感分类。
3. 评论与弹幕分流。
4. 规则层纠偏。
5. 切面召回。
6. `aspect + text` 二次情感判别。
7. 时间轴聚合与写库。

### 9.3 当前输出字段

评论与弹幕分析结果可输出：

1. `label`
2. `score`
3. `confidence`
4. `intensity`
5. `source`
6. `version`
7. `emotion_tags`

评论额外支持：

1. 多切面识别结果 `aspect_details_json`
2. 兼容旧逻辑的主切面 `aspect`

### 9.4 规则层作用

规则层并不替代模型，而是：

1. 对低置信度结果做小范围纠偏。
2. 检测网络语和情绪标签。
3. 弥补部分模型对短文本与口语文本的不足。

### 9.5 评估与回填能力

项目已配套：

1. 单任务重算脚本。
2. 批量回填脚本。
3. 低置信度样本导出脚本。
4. 标注候选集构建脚本。

这些能力有助于后续论文中“实验设计”和“模型迭代优化”章节撰写。

## 10. 当前仓库实现状态

### 10.1 已完成内容

1. 前后端和 Python 微服务主体架构已经打通。
2. 用户认证、项目管理、任务提交、热门榜单、扫码绑定、结果查询均已实现。
3. Transformer 情感分析链路已接入。
4. Java 实体与 Mapper 已扩展至新字段。
5. 历史任务支持回填和时间轴重建。
6. 热门视频刷新逻辑已调整为成功时重建 100 条数据。

### 10.2 当前仍需注意的边界

1. 切面筛选虽然已支持通过 `aspect_details_json` 查询，但依赖 MySQL JSON 搜索，数据量增大后需要关注性能。
2. 热门视频的“100 条保障”依赖在线热门接口返回足够唯一视频，若平台侧数据不足会保留旧数据并返回失败。
3. 情感分析已较初始方案明显改进，但仍不是领域精调后的最佳效果，后续仍可继续构建标注集并做微调。
