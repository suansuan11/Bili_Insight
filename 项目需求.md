# Bili-Insight智能舆情分析平台 

## 1. 项目用处 (目标用户与核心价值)

### 1.1 目标用户一：UP主 / MCN机构

·    **痛点:** 视频复盘耗时，无法精确定位观众反馈；缺乏数据支持选题。

·    **核心价值 (视频复盘助手 & 选题灵感库):**

1. **交互式复盘:** 通过“播放器+情绪时间轴”联动，精确定位到秒，识别“高光时刻”与“败笔之处”。
2. **趋势发现:** 追踪热门排行榜数据，为内容创作提供灵感。

### 1.2 目标用户二：品牌方 / 市场部

·    **痛点:** 难以实时、量化地掌握B站新品口碑或KOL投放效果。

·    **核心价值 (B站口碑监测雷达):**

1. **自动化舆情监测:** 创“建监测项目”后，系统自动、持续地抓取和分析相关视频，提供数据报表。
2. **口碑切面分析:** 量化分析用户对产品**不同维度**（如：外观、性能、价格）的情感倾向。

## 2. 核心功能模块 (Functional Requirements)

### 模块一：SaaS平台核心 (用户与项目管理)

1. **用户认证系统:**

o  功能：提供用户注册、登录、登出功能。

o  技术：**Java Spring Security + JWT** 实现API鉴权。

2. **角色权限:**

o  功能：区分“个人创作者”和“企业品牌方”两种角色，后者开放“项目监测”功能。

3. **项目监测管理 (品牌方核心):**

o  功能：允许品牌方用户创建、编辑、删除“舆情监测项目”。

o  配置：项目需配置监测关键词（如："XX手机"）、目标UP主列表等。

o  数据：所有项目和分析结果均与用户ID绑定，实现数据隔离。

### 模块二：数据分析任务中心 (交互入口)

1. **热门排行榜:**

o  功能：展示B站实时热门排行榜单，点击可提交分析。

2. **单一视频复盘 (UP主核心):**

o  功能：提供输入框，用户粘贴B站视频URL或BVID，提交**一次性**的深度分析任务。

3. **UP主追踪:**

o  功能：输入UP主MID，系统自动分析该UP主最新发布的N个视频数据表现。

4. **监测项目仪表盘 (品牌方核心):**

o  功能：展示用户创建的“监测项目”的宏观看板，汇总该项目下所有视频的总体情感趋势、热门切面等。

### 模块三：交互式视频复盘页 (项目最大亮点)

1. **视频播放器:**

o  功能：内嵌Bilibili视频播放器。

2. **情绪-时间联动曲线 (ECharts):**

o  功能：在播放器下方，绘制一条与视频时间轴完全对应的“情绪折线图”（Y轴为情感均值）。

o  **交互:** 点击折线图上任意时间点（如3:15），视频播放器自动跳转到3:15。

3. **切面情感分析 (ABSA) (AI核心):**

o  功能：取代简单的词云。展示预设的几个“分析切面”（如：`外观`, `性能`, `续航`, `价格`）及其对应的情感评分（如：`外观: 90% 正面`）。

o  **交互:** 点击某个切面（如“续航”），下方的评论列表自动筛选出所有讨论续航的评论。

4. **智能评论筛选:**

o  功能：提供按情感（正面/负面）、按切面、按热度的评论筛选和查看功能。

## 3. 架构与技术实现

本方案采用“**Java****业务后端 + Python分析微服务**”的解耦架构。

### 模块四：业务后端 (Java Spring Boot)

·    **定位:** 系统的“主脑”，负责除NLP分析外的所有业务逻辑。

·    **技术:** Spring Boot + Spring Data JPA/MyBatis + Spring Security + MySQL。

·    **核心职责:**

1. **API提供:** 负责模块一、二、三中所有前端页面的API数据接口。
2. **服务调用:** 作为**调用方**。当需要分析数据时，使用 `**RestTemplate**` **或 `WebClient`** 向Python微服务发送HTTP请求。
3. **异步任务:**

§ 为避免前端长时间等待，分析任务采用异步处理。

§ **流程:** Java收到前端分析请求 -> Java向Python服务发起**异步**HTTP调用（或Java先存任务到DB，Python再来取） -> Java**立即**返回“任务已提交”给前端 -> 前端通过轮询Java的“任务状态”接口来等待结果。

4. **定时任务 (`@Scheduled`):**

§ 任务1：定时更新“热门排行榜”数据。

§ 任务2：**核心** - 定时触发“项目监测”，扫描所有用户项目，搜索新视频，并调用Python服务对新视频进行分析。

### 模块五：分析微服务 (Python FastAPI)

·    **定位:** 系统的“分析引擎”，负责所有爬虫和NLP计算。

·    **技术:** FastAPI + Scrapy/Requests + Jieba + SnowNLP。

·    **核心职责:**

1. **API实现:** 作为**被调用方**。提供清晰的HTTP API，如 `POST /api/analyze/video`，接收Java传来的BVID或关键词。
2. **数据爬取:** 内置爬虫逻辑，获取视频信息、评论、弹幕。
3. **NLP情感分析 (`SnowNLP`):**

§ 对每一条评论和弹幕进行情感打分（0-1）。

4. **轻量级ABSA算法 (巧思):**

§ **实现:**

§ a. 预先定义切面及其关键词库 (如: `ASPECTS = {"``续航": ["续航", "电池", "没电"]}`）。

§ b. 遍历所有评论，通过关键词匹配将评论归类到不同“切面”。

§ c. 计算每个切面下所有评论的 `SnowNLP` 情感均值，形成分析结果。

5. **数据入库:**

§ Python服务在完成所有分析后，将结构化的分析结果（JSON格式的情绪时间轴、ABSA结果等）**直接写入MySQL数据库**的相关表中，供Java后端读取。
