# Bili-Insight 智能舆情分析平台 项目需求与实现说明

## 1. 项目概述

### 1.1 项目名称

Bili-Insight 智能舆情分析平台

### 1.2 项目定位

本项目面向 Bilibili 内容生态，构建一个集用户管理、视频分析、热门发现、账号绑定、项目监测与可视化展示于一体的舆情分析平台。系统采用“前端展示层 + Java 业务后端 + Python 分析微服务”的分层架构，实现对 B 站视频评论与弹幕数据的采集、情感分析、切面分析、时间轴聚合和结果展示。

### 1.3 目标用户

1. UP 主与 MCN 机构
   - 关注视频复盘、观众反馈定位、内容优化与选题发现。
2. 品牌方与市场部门
   - 关注新品口碑、投放效果、项目监测和负面舆情预警。
3. 数据分析类课程与毕业设计场景
   - 适合展示“分层架构设计 + 异步任务处理 + 文本情感分析 + 可视化交互”完整工程链路。

### 1.4 核心价值

1. 提供视频级舆情分析能力，实现从 BVID 提交到结果展示的完整闭环。
2. 通过情绪时间轴定位视频高光与争议片段，支持复盘分析。
3. 通过切面情感分析量化“外观、性能、价格、内容、主播”等维度反馈。
4. 提供项目化监测能力，支持用户对指定视频集合持续跟踪。
5. 提供热门视频榜单，为内容发现和分析入口提供数据支持。

## 2. 总体需求

### 2.1 功能需求

系统应具备以下核心功能：

1. 用户注册、登录、JWT 鉴权与个人信息查询。
2. Bilibili 账号扫码绑定、登录状态检测、当前用户信息获取、登出。
3. 单视频分析任务提交、任务进度查询、结果查询。
4. 评论列表、弹幕列表、时间轴与切面结果查询。
5. 热门视频抓取、状态查询与列表展示。
6. 监测项目的创建、更新、删除、查询和统计。
7. Dashboard 总览统计，包括任务数量、情感分布、切面排行和趋势信息。
8. 评论导出。

### 2.2 性能与工程需求

1. 分析任务提交后应快速返回任务受理结果，不阻塞前端页面。
2. Python 分析过程应以后台异步方式执行，支持任务状态轮询。
3. 热门视频抓取任务应在成功执行时保证热门视频表中存在 100 条数据。
4. 评论、弹幕和时间轴等结果应持久化到 MySQL，支持后续查询与回溯。
5. 系统配置应尽量通过 `application.yml` 与 `.env` 管理，避免硬编码敏感信息。

## 3. 系统架构需求

### 3.1 架构模式

系统采用三层解耦架构：

1. 前端：Vue 3 单页应用，负责交互与可视化。
2. Java 后端：Spring Boot 业务后端，负责用户、权限、项目、任务编排、接口聚合。
3. Python 微服务：FastAPI 分析服务，负责 B 站数据抓取、情感分析、时间轴生成与结果写库。

### 3.2 技术选型

前端技术栈：

1. Vue 3
2. Vue Router
3. Axios
4. Element Plus
5. ECharts
6. Vite

Java 后端技术栈：

1. Spring Boot 2.7
2. Spring Security
3. JWT
4. MyBatis
5. MySQL

Python 微服务技术栈：

1. FastAPI
2. bilibili-api-python
3. PyMySQL + DBUtils
4. Transformers
5. Torch
6. SnowNLP
7. PyYAML
8. pandas / scikit-learn / tqdm

## 4. 业务模块需求

### 4.1 用户与认证模块

应支持：

1. 用户注册。
2. 用户登录。
3. JWT 令牌签发与前端持久化。
4. 登录态校验与过期处理。
5. 获取当前登录用户信息。

### 4.2 Bilibili 账号绑定模块

应支持：

1. 生成 B 站扫码登录二维码。
2. 轮询扫码状态。
3. 扫码成功后将 `SESSDATA`、`bili_jct`、`buvid3` 保存到当前平台用户记录。
4. 检测当前 B 站登录状态。
5. 获取当前绑定的 B 站用户信息。
6. 登出并清空绑定凭证。

### 4.3 视频分析任务模块

应支持：

1. 输入 BVID 提交分析任务。
2. 任务创建后写入 `analysis_task` 表。
3. Java 后端通过 HTTP 调用 Python 分析接口。
4. Python 后台完成视频信息、评论、弹幕和时间轴分析。
5. 前端通过任务状态接口轮询进度与当前步骤。

### 4.4 热门视频模块

应支持：

1. 后台抓取热门视频数据。
2. 清空旧热门数据并重建新数据。
3. 保证热门视频表中保留 100 条热门视频记录。
4. 前端展示热门视频列表，并可直接跳转到分析页面。

### 4.5 项目监测模块

应支持：

1. 创建监测项目。
2. 修改监测项目。
3. 删除监测项目。
4. 查询当前用户下的项目列表与详情。
5. 查询项目统计信息。

### 4.6 交互式复盘模块

应支持：

1. 视频分析结果详情页展示。
2. 评论列表展示与筛选。
3. 弹幕列表展示与筛选。
4. 情绪时间轴展示。
5. 切面情感分析结果展示。
6. 点击时间轴后辅助定位视频片段。

## 5. 数据分析需求

### 5.1 数据采集需求

系统需要采集以下数据：

1. 视频基础信息。
2. 评论数据。
3. 弹幕数据。
4. 热门视频榜单数据。

### 5.2 情感分析需求

系统当前已从传统 SnowNLP 方案升级到 Transformer 为主、规则层纠偏为辅的混合方案，需满足：

1. 输出情感标签：`POSITIVE`、`NEUTRAL`、`NEGATIVE`。
2. 输出情感分数，范围为 `[-1, 1]`。
3. 输出情感置信度。
4. 输出情感强度：`WEAK`、`MEDIUM`、`STRONG`。
5. 输出来源标识与模型版本。
6. 输出情绪标签，如 `sarcasm`、`complaint`、`praise`。

### 5.3 切面分析需求

系统需要支持：

1. 基于关键词的切面召回。
2. 基于 `aspect + text` 的二次情感判别。
3. 单条评论支持多个切面结果。
4. 将结果写入 `aspect_details_json` 字段。

### 5.4 时间轴需求

系统需要：

1. 基于弹幕时间点进行分桶聚合。
2. 结合情感分数与置信度生成时间轴。
3. 支持生成切面聚合 JSON。
4. 对任务级时间轴执行覆盖更新。

## 6. 数据存储需求

### 6.1 核心表

当前系统核心数据表包括：

1. `user`
2. `analysis_task`
3. `project`
4. `video_info`
5. `video_comment`
6. `video_danmaku`
7. `sentiment_timeline`
8. `popular_videos`

### 6.2 扩展字段需求

评论与弹幕表应支持以下扩展字段：

1. `normalized_content`
2. `text_type`
3. `sentiment_confidence`
4. `sentiment_intensity`
5. `sentiment_source`
6. `sentiment_version`
7. `emotion_tags_json`
8. `aspect_details_json`（评论）

时间轴表应支持：

1. `timeline_version`
2. `aggregation_meta_json`

## 7. 当前实现情况

### 7.1 已实现

当前仓库已完成以下主要功能：

1. Vue 前端主页面、登录注册页、Dashboard、热门视频页、分析任务页、分析详情页、项目页和设置页。
2. Spring Security + JWT 鉴权。
3. 用户注册、登录、当前用户查询。
4. B站账号扫码绑定、轮询、状态检查、当前用户信息获取、登出。
5. Java 到 Python 的 HTTP 通信。
6. 分析任务提交、轮询、结果查询。
7. Python 后台抓取评论、弹幕和热门视频。
8. Transformer 情感分析、规则纠偏、切面分析、时间轴生成。
9. 评论、弹幕、时间轴和热门视频写入 MySQL。
10. 热门视频抓取逻辑已调整为成功时重建 100 条数据。
11. 情感分析评估脚本、回填脚本和候选标注集导出脚本。

### 7.2 当前适合作为毕业设计展示的内容

1. 多技术栈协同架构设计。
2. JWT 鉴权与扫码绑定流程。
3. 异步任务编排与轮询机制。
4. 中文短文本情感分析方案升级。
5. 切面级情感分析与时间轴聚合。
6. 前端可视化展示与业务闭环实现。

## 8. 运行要求

### 8.1 环境要求

1. JDK 8
2. MySQL 8 或兼容版本
3. Node.js
4. Python 3.14 虚拟环境

### 8.2 启动方式

1. Java 后端：`mvn spring-boot:run`
2. Python 服务：`.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8001 --log-level info`
3. 前端：`npm run dev`
