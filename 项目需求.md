# Bili-Insight 智能舆情分析平台 项目需求

## 1. 项目用处 (目标用户与核心价值)

### 1.1 目标用户一：UP主 / MCN机构
- **痛点:** 视频复盘耗时，无法精确定位观众反馈；缺乏数据支持选题。
- **核心价值 (视频复盘助手 & 选题灵感库):**
  1. **交互式复盘:** 通过“播放器+情绪时间轴”联动，精确定位到秒，识别“高光时刻”与“败笔之处”。
  2. **趋势发现:** 追踪热门排行榜数据，为内容创作提供灵感。

### 1.2 目标用户二：品牌方 / 市场部
- **痛点:** 难以实时、量化地掌握B站新品口碑或KOL投放效果。
- **核心价值 (B站口碑监测雷达):**
  1. **自动化舆情监测:** 创建“监测项目”后，系统自动、持续地抓取和分析相关视频，提供数据报表。
  2. **口碑切面分析:** 量化分析用户对产品**不同维度**（如：外观、性能、价格）的情感倾向。

## 2. 核心功能模块 (Functional Requirements)

### 模块一：SaaS平台核心 (用户与项目管理)
1. **用户认证系统:**
   - 功能：提供完善的用户注册和登录流程。
   - 技术：基于 **Java Spring Security + JWT** 实现API鉴权与安全访问控制。
2. **角色权限:**
   - 功能：区分不同类型的用户，保证数据和操作的安全性。
3. **项目监测管理:**
   - 功能：允许用户创建、编辑、删除“舆情监测项目”，项目与用户ID绑定实现数据隔离。

### 模块二：数据分析任务中心 (交互入口)
1. **热门排行榜:**
   - 功能：展示B站实时热门排行榜单，点击可直接提交分析。
2. **单一视频复盘 (UP主核心):**
   - 功能：输入BVID或URL，提交深度的视频舆情分析任务。
3. **监测项目仪表盘 (品牌方核心):**
   - 功能：展示用户创建的监测项目概览。

### 模块三：交互式视频复盘页 (Vue 核心端实现)
1. **视频播放器与视图 (`VideoReviewView`):**
   - 功能：内嵌 Bilibili iframe 视频播放器。
2. **情绪-时间联动曲线 (ECharts):**
   - 功能：播放器下方绘制与视频时间轴对应的“情绪折线图”。支持点击折线图上的时间点进行视频跳转联动。
3. **切面情感分析 (ABSA) 与评论筛选 (`CommentList`):**
   - 功能：展示分析切面（如`外观`, `性能`等）及其情感评分。支持按情感（正/负面）和按切面（ABSA）对评论列表进行精准过滤与展示。

## 3. 架构与技术实现

本方案采用“**Java业务后端 + Python分析微服务 + Vue前端**”的解耦架构。

### 模块四：业务后端 (Java Spring Boot)
- **定位:** 系统的“主脑”，负责除NLP分析外的所有业务逻辑及用户状态管理。
- **核心职责:**
  1. **API与鉴权:** 提供 RESTful API 接口，并负责 JWT 令牌分发与请求拦截。
  2. **服务调用 (`PythonApiClient`):** 采用 HTTP REST 调用替换了低效的子进程执行方案，通过网络与 Python 服务通讯。
  3. **异步任务:** Java 接收前端请求后，向 Python 服务发起异步 HTTP 调用，立即返回任务受理状态，前端通过轮询获取最新进度。

### 模块五：分析微服务 (Python FastAPI)
- **定位:** 系统的“分析引擎”，负责高并发爬虫和 NLP 计算。
- **核心优化:** 修复了异步后台阻塞和重复路由问题；敏感配置全量迁移至 `.env`。
- **核心职责:**
  1. **API实现:** 提供 FastAPI 接口接收 Java 端的分析请求。
  2. **爬虫与NLP:** 获取视频弹幕、评论，利用 SnowNLP进行0-1情感打分及基于规则的 ABSA 切面分析。
  3. **直接数据入库:** 分析完成后产生的大批量数据（评论、弹幕、时间轴JSON）**直接写入 MySQL**，避免跨服务大文件传输阻塞。
